About Jupyter¶

What is this thing?

Jupyter combines interwoven text, data, and code, in a format that runs in a web browser.

  • ‘Jupyter’ = JUlia, PYthon, and R - but it’s really language-agnostic

  • Jupyter Notebooks let you run code immediately

  • Jupyter Notebooks can connect to a server that has the right environment/dependencies to execute the code successfully. In this case, we’re using a server provided by ITHAKA, parent company to JSTOR/Portico.

Jupyter Notebooks are collections of cells that do different things.

You can navigate through the cells of a notebook (including this one) by using the ‘run’ command in the toolbar above.

What do the cells do? For this exercise, we’ll talk about 2 kinds of cells.

  • This cell is an example of a markdown cell. Markdown cells just contain text, and are used for commenting code, creating headings, etc.

  • There are also executable code cells.

    • How does this magic happen? There’s a kernel, or computational engine, that runs the code inside your Notebook. In this case our kernel = Python 3, as you can see in the top right corner of this page under the ‘logout’ button.

Let’s try running a simple code cell to see what happens. Don’t worry, you can’t break it! If you get stuck you can always reset the notebook by resetting the kernel (the ‘refresh’ symbol in the toolbar).

Modify the cell below by:

  1. Selecting the cell. The cell will be highlighted in blue when selected, then will change to green when you begin to edit it

  2. Delete the text that says ‘yourname’ (but keep the ’ ‘marks)

  3. Type your name inside the ’ ‘s instead, and finally

  4. Hit ‘Run’ in the toolbar above.

def print_hey(recipient):
    return 'Hey, {}!'.format(recipient)
print_hey('yourname')

As you can see, when you run a code cell the output appears below it in another cell. The output after you run a cell essentially forms part of the document as you work through it.

You may have noticed the In [ # ] and Out [ # ] to the left of the cells above. The notebook will number functions in the order that you run the cells. In general, you’ll want to run cells in sequential order as you work through a notebook to avoid errors.

Finally, Jupyter shows that a cell is currently running by displaying an asterisk in the brackets like so: [ * ] The small circle next to ‘Python 3’ at the top right of the page also gives insight into the status of the kernel; hover over it for a tooltip if you’re not sure what state it’s in.
